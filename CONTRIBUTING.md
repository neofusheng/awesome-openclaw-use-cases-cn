# Contributing to Awesome OpenClaw Use Cases CN

感谢你愿意补充 OpenClaw 的真实案例。

这个仓库优先欢迎：

- 真实跑通过的案例
- 有明确场景、步骤、结果的案例
- 有截图 / 配置 / Prompt / 工作流的案例
- 能让别人快速复用的案例

## 你可以贡献什么

- 新的 use case
- 优化现有案例
- 补充截图/GIF/演示链接
- 补充提示词、配置、踩坑记录
- 增加英文标题/关键词，帮助全球用户搜索到

## 提交建议

每个案例尽量包含：

1. 场景
2. 用户对象
3. 使用渠道
4. 用到的 OpenClaw 能力
5. 工作流步骤
6. Prompt / 配置
7. 结果 / 效果
8. 踩坑
9. 截图 / 演示

## 文件组织建议

建议在 `usecases/` 下新增一个 markdown 文件：

```text
usecases/
  personal-assistant-feishu.md
  content-repurpose-bot.md
  github-pr-review-agent.md
```

文件名尽量：

- 全小写
- 用英文或拼音
- 用 `-` 连接
- 一眼能看懂主题

## PR 标题建议

```text
feat: add feishu personal assistant use case
fix: improve README structure
docs: add contribution template
```

## 质量标准

会优先合并这类内容：

- 可复制
- 信息密度高
- 结构清楚
- 有传播亮点

不建议：

- 纯灵感、无细节
- 只有一句话介绍
- 和 OpenClaw 关联很弱

## 如果你懒得整理

没事，直接提 Issue 也行。

只要告诉我：

- 你做了什么
- 解决了什么问题
- 效果怎么样
- 大概怎么做的

我可以帮你整理成标准案例。

# 日报 / 周报 Agent：自动把碎片进展整理成可汇报内容

- 场景：从聊天、文档、任务记录中提炼周报/日报
- 用户：产品经理、运营、顾问、团队成员
- 渠道：Feishu / Discord / Telegram / CLI
- 用到的能力：read / feishu_doc / message / memory / cron
- 难度：中级
- 效果：减少机械整理时间，让复盘和汇报更稳定

## 为什么值得做

多数人不是没做事，而是工作痕迹散落在各处。Agent 最适合干这种“收集碎片 → 结构化输出”的事。

## 工作流

1. 读取最近一周的任务/对话/文档
2. 提取完成项、风险项、下一步
3. 输出不同风格版本：内部汇报 / 向上汇报 / 对客户汇报
4. 自动写入文档或发到指定渠道

## 踩坑

- 周报最怕空泛，必须要求输出“结果 + 证据 + 下一步”
- 最好先定义固定模板，避免每周口径不一致

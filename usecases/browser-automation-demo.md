# Browser 自动化 Demo：让 Agent 不只是会说，还会点

- 场景：让 OpenClaw 在网页里执行点击、填写、提取信息等动作
- 用户：自动化爱好者、运营、研究员、开发者
- 渠道：Web / Browser Relay
- 用到的能力：browser / snapshot / act / tools
- 难度：中级
- 效果：把“AI 建议你做什么”升级成“AI 直接帮你做一部分”

## 为什么值得做

浏览器自动化是最容易做出 wow effect 的演示之一，因为结果非常直观。

## 工作流

1. 打开目标页面
2. 快照页面元素
3. 定位按钮/输入框
4. 执行动作并提取结果
5. 根据结果继续下一步

## 踩坑

- UI 经常变化，稳定性要靠更好的定位方式
- iframe/canvas 页面会显著更难自动化
- 演示时要挑流程简单、反馈明确的页面

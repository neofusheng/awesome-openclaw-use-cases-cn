# 内容改写工作流：一篇长文拆成多平台内容

- 场景：把一篇文章快速改写成公众号摘要、朋友圈文案、短帖、标题备选
- 用户：内容创业者、顾问、IP、运营
- 渠道：Feishu / Telegram / Web
- 用到的能力：message / doc read-write / structured prompting / memory
- 难度：入门
- 效果：提升内容复用率，降低二次创作成本

## 为什么值得做

大多数人不是不会写，而是没时间把一篇内容拆成多个版本。

OpenClaw 很适合做“内容中台”：

- 读原文
- 判断受众
- 生成不同平台版本
- 保留你的语气与定位

## 工作流

1. 把原文丢给 Agent
2. 指定目标平台与目标读者
3. 让 Agent 输出多个版本：短版 / 金句版 / 标题版 / 行动号召版
4. 让 Agent 继续迭代到可发布状态

## 关键 Prompt / 配置

```txt
把这篇文章拆成 4 个版本：
1. 朋友圈短文案
2. 公众号导读
3. 即刻/推特短帖线程
4. 3 个更有传播力的标题
要求：保留原意，但更利于传播与转发。
```

## 踩坑

- 不要只说“帮我改写”，要明确平台、受众、风格、长度
- 最好沉淀成固定模板，否则每次输出风格不稳定
